    05

    Boundary

    what this demonstration does not claim

    Honesty boundary. This standalone page is the intake acceptance harness. It does not mint Genesis shards or sign with axm-hybrid1. Surfaces that pass here are candidates for custody, not truth — promotion into sealing and detached verification happens in the AXM kernel, after intake.

    Standing rule. A missing field does not kill the demonstration. Missing fields lower the evidence tier or create a declared gap. Human review routes attention; it does not convert weak evidence into stronger evidence.

    06

    The system behind this page

    every property below is live — all demo data synthetic; proven, not deployed
    1. ↳GhostBox — the intelligence spoke: watch the incumbent, stage the switch. its exit demo shows an export leaving a vendor silo and sealing into a sovereign record; the Ship of Theseus shows a nine-plank vendor workflow sealed at each plank's honest tier
    2. ↳axm-genesis — the frozen kernel underneath everything: sealed shards signed Ed25519 ‖ ML-DSA-44, verifiable offline against the record's own bytes and an out-of-band key. this is the custody path surfaces are promoted into after intake
    3. ↳axm-core — the hub: the Spectra query runtime, SQL over sealed shards with the custody gate in the read path.
    4. ↳axm-console — the operator cockpit: one CLI drives capture → seal → verify → ask and returns receipts.
    5. ↳axm-fleet — the fleet-lifecycle spoke, pinned to the v1 kernel. axm-sfn, the hardware-custody spoke, is its supply-side sibling
    6. ↳axm-aide — the personal spoke: custody of memory, with human review routing attention rather than adjudicating truth.

    If a finding above does not match operational reality when run against your own exports, that mismatch is the most valuable output this page can produce — it identifies exactly which record, field, or timestamp is misrepresenting the physical state.
