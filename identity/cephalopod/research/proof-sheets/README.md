# Proof-sheet ledger

These images are exploratory design evidence. They are not canonical pixel maps, sanctioned exports, or a substitute for direct hand-setting at 32 × 32, 24 × 24, 16 × 16, and 12 × 12.

The repository contains four controlled proof-sheet previews, each with four candidates:

1. `01-organism-topology.webp` holds corruption relatively constant while testing control, anchor plus emergence, anchor plus distributed action, and distributed emergence. SHA-256: `7ce877b169ec7c22278cd215efa41e5764fb0c8fa2b26f06eaccad8cc6f469ec`.
2. `02-corruption-mechanics.webp` holds the organism comparatively stable while testing row displacement, tile duplication, missing-tile apertures, and palette or layer overwrite. SHA-256: `a1fe89c2faef987cb02a0b1f1ba8814f07cb15ac9c31a00026608bf2c15eeb00`.
3. `03-failure-coupling.webp` tests bridge, split route, brace and redirect, and coherent re-emergence. Every fault is required to create a usable advantage. SHA-256: `ce2a5f8681d8af7fa6b817340722cc23efcf920e66c94c3f3e2afb0d67a4e354`.
4. `04-integrated-candidates.webp` spans anchor dominance, substrate dominance, distributed intelligence, and reroute dominance. SHA-256: `3b3cf0850a86fb4de75ffd3c11bbd01d6e45c62e53b5ae2053b2c6977b1ca9e9`.

The previews are 320 × 169 WebP files cropped from the generated four-sheet matrix after removing an unreliable generated reference strip. They preserve the comparative layout for repository review while keeping the external research sources out of the visual source lineage. No pixel in these previews has source authority. Any candidate selected for development must be rebuilt as an explicit bitmap map.
